# Contributing

Contributions are welcome. Here's how to get started.

## Reporting issues

Open an issue on the [Codeberg repository](https://codeberg.org/melsner/lxmf_group). Include:

- What you expected to happen
- What actually happened
- Steps to reproduce
- Log output (run with `--loglevel DEBUG` if possible)

## Development setup

See [docs/development.md](docs/development.md) for instructions on setting up the dev environment, running tests, and building.

## Submitting changes

1. Fork the repository
2. Create a branch for your change
3. Make your changes
4. Run the tests: `make test`
5. Submit a pull request

## Code style

- Keep it simple and readable
- Follow existing patterns in the codebase
- Add tests for new functionality when practical
